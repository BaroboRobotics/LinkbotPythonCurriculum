Connecting to the Robot and Moving Motors

Lets write a program to move a Linkbot. To do this, we must first establish a way for our Python programs to connect to particular Linkbots to control. The easiest way to do this is through BaroboLink.

To begin, open BaroboLink and connect to the robot you want to control. When you write your Python program, the Python program you write will automatically communicate with BaroboLink and run on the robot currently connected to BaroboLink.

Then, execute the following python program:

Example 4.5. A Python program that moves both joints an a Linkbot‑I.

#!/usr/bin/env python
#Filename: move.py

from barobo import Linkbot

linkbot = Linkbot()
linkbot.connect()

print ("Moving both joints 90 degrees...")
linkbot.move(90, 0, 90)


Lets take a look at the previous program in detail. The first two lines, 

#!/usr/bin/env python
#Filename: move.py

contain the same type of boilerplate code we saw in our previous example. The next line reads: 

from barobo import Linkbot

This piece of code is necessary to begin controlling Linkbots. In Python, you can use the "import" command to add additional functionality to your python programs. There are many packages available to import into Python provided by Python and also third-party organizations. Some packages help you access system resources, some packages perform numerical analysis and display graphical plots, etc. This line accesses the "barobo" package, which is a package that helps control Barobo robots. Inside the barobo package, there is a part called "Linkbot". Thus, the line above "imports the Linkbot portion of the barobo package". This line of code can be considered boilerplate code since most programs that control Linkbots will have this line, or a line similar to this line of code.
